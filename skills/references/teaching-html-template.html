01

什么是死锁

Definition
Definition
多个进程互相等待对方持有的资源
导致全部进程都无法继续推进的状态。
类比 ▸ 四车路口堵死, 谁都不动
Example
P1 持打印机等扫描仪 · P2 持扫描仪等打印机
两个进程互相持有对方需要的资源, 又互相等待对方释放, 永远等下去。
Diagram · 死锁环路 (RAG 资源分配图)
P1 / P2 互相持有对方所需资源 → 形成等待环

04

银行家算法

Banker's Algorithm · 必考
Data Structures
4 个核心数据结构
Available
系统当前可用资源向量
Max
每进程最大需求矩阵
Allocation
每进程已分配资源矩阵
Need
还需要资源 = Max − Allocation
Algorithm
安全性算法步骤
01
找进程 Pi 使 Need[i] ≤ Available
02
假设 Pi 完成, 回收资源
Available += Allocation[i]
03
重复直到全部完成 → 安全
否则 → 不安全
05

资源分配图 + 概念辨析

RAG & Comparison
Tool
资源分配图判断死锁
符号
进程 = 圆圈 ○    资源 = 方框 □
判断规则
单实例资源: 有环 = 死锁
多实例资源: 有环 ≠ 一定死锁
死锁 → 一定有循环等待
循环等待 → 不一定死锁
Comparison
死锁 vs 活锁 vs 饥饿
死锁Deadlock
进程都阻塞, 无法推进
面对面走廊都不让
活锁Livelock
不阻塞但都做无效操作
同时让又同时让回
饥饿Starvation
长期得不到资源
低优先级一直被抢
Pitfall · 高频易错
这些说法都是 错的
  • "死锁 = 循环等待" ✓ 正确: 循环等待是必要不充分条件
  • "有环就一定死锁" ✓ 正确: 多实例资源有环不一定死锁
  • "死锁可以彻底消除" ✓ 正确: 只能预防/避免/检测, 不能消除
