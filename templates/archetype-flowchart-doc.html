每步细节

1 · git push .git/hooks

触发 webhook 到 CI 系统。push 到 main / release/* 进入 prod pipeline, 其他分支只跑 CI。

2 · CI lint + test .github/workflows/ci.yml

串行: ESLint → TypeScript → Jest (unit) → Playwright (E2E)。任意失败 fail-fast。

3 · Image build Dockerfile

多阶段构建, 缓存 npm install 层。镜像 push 到 GHCR, tag 是 commit SHA。

4 · Canary 5% → SLO gate k8s/ingress.yaml

5% 流量打到新版本, 观察 5 分钟 error rate < 0.5% / p95 < 1s。任何违反 → auto-rollback。

5 · Promote 100% k8s/ingress.yaml

SLO gate 通过, 流量比例递增到 100% (25% → 50% → 100%, 每档 2 分钟)。

Rollback 触发条件

⚠
下面任何一条触发自动回滚, 不需要人为决策 — pipeline 自带"快速止血"能力。
  • Error rate > 0.5% sustained 60s
  • p95 latency > 1s sustained 120s
  • Health check failure > 30%
  • Smoke test 失败 (合成请求)
