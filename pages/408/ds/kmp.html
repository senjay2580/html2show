01

什么是 KMP

Definition
Problem
字符串匹配: 在文本串 T 中查找模式串 P 首次出现的位置
朴素算法 O(nm); KMP O(n+m), 核心是利用模式串自身的重复结构, 让主串指针永不回退。
类比 ▸ 看错考卷不必从头看起, 跳过已确认的部分
Example
T = "ABABABCABAB" · P = "ABABC"
朴素: 在 T[4]='A' 与 P[4]='C' 失配后, 把 i 退回 1 重新匹配。
KMP: i 不动, 让 j 跳到 next[4]=2 继续匹配, 一次扫描即可。
Diagram · KMP 自动机示意 (P = "ABABC")
实线 = 字符成功匹配前进 · 虚线 = 失配回退

王道老版
下标从 1 开始
next[1] = 0
next[j] = (P[1..j-1] 中最长相等真前后缀长度) + 1
考点 ▸ 失配位置直接读 next[j] 跳转
现代标准版
下标从 0 开始 (前缀函数 π)
π[0] = 0
π[i] = P[0..i] 中最长相等真前后缀长度
两版差 1, 求法和效果一致
本质
next[j] 记录的是 "失配时模式串能少跳几步"
等价于 P[0..j-1] 这段已匹配子串的最长自相似度

KMP 匹配算法

Main Algorithm · 必考
Variables
两个指针 + 一张前缀表
i
主串 T 指针 · 永不回退
j
模式串 P 指针 · 失配时跳到 next[j]
next[]
长度 m 的前缀函数表
复杂度
O(n + m) · 摊还分析保证
Main Loop
三步循环走到底
01
比较 T[i] 与 P[j]
02
相等 → i++, j++
不等 → j = next[j] · i 不变
03
j 退到 0/-1 起点 → i++, j = 0
j == m → 匹配成功, 返回 i − m
摊还证明
主串 i 最多 +1 进 n 次 · j 最多被回退 n 次 (因每次 j 增加才能被回退)
合计 O(n + m) · 与朴素 O(nm) 形成数量级差距

nextval 改进数组

Optimized · 王道必考
Why nextval ?
next 数组在某些模式下会"白跳一次"
考虑 P = "aaaab"。当在第 4 位 'a' 处失配时, next 让 j 跳到 3, 但 P[3] 仍是 'a', 与刚才失配的字符相同, 必然再次失配, 浪费一次比较。 nextval 把这种"必然失配"的跳转一次性合并到底, 直接跳到不同字符的位置或起点。
Rule
nextval 的递推规则
对每个位置 j (j ≥ 2)
若 P[j] == P[next[j]]
  → nextval[j] = nextval[next[j]]
否则
  → nextval[j] = next[j]
初值 nextval[1] = 0
逐位向后推, 一遍扫完即可
Example · P = "aaaab"
完整推导对比表 (王道老版下标)
j
1   2   3   4   5
P[j]
a   a   a   a   b
next
0   1   2   3   4
nextval
0   0   0   0   4
前 4 位都是 'a', 失配时连续合并到 0; 第 5 位 'b' 与 P[next[5]]='a' 不同, 保留 next 值 4。
Pitfall · nextval 高频易错
这些说法都是 错的
  • "nextval 严格优于 next" ✓ 正确: 仅在失配点字符相同时少几次比较, 最坏时间复杂度同为 O(n+m)
  • "nextval 可以替代 next 用在主算法里" ✓ 正确: 主算法可直接用 nextval 表跳转, 通常更快
  • "nextval 公式里 next 也要先改成 nextval" ✓ 正确: 递推时仍读 next[j] 找跳转目标, 仅赋值用 nextval[next[j]]

完整匹配过程 · 一步一步看

Walkthrough · T="ABABABCABAB" P="ABABC"
Setup
已先求出 next 表 (下标 0 起): next = [0, 0, 0, 1, 2]
下面跟踪每一步的 (i, j), 红色加粗表示当前比较位置, 失配时 j 跳, i 不动。
Trace
01
i=0 j=0 · T[0]='A' = P[0]'A' → 匹配, 同步 +1
02
i=1..3 j=1..3 · 连续匹配 "ABAB"
03
i=4 j=4 · T[4]='A' ≠ P[4]'C' → 失配! j = next[4] = 2, i 不动
04
i=4 j=2 · T[4]='A' = P[2]'A' → 匹配, 同步 +1
05
i=5 j=3 · T[5]='B' = P[3]'B' → 匹配, 同步 +1
06
i=6 j=4 · T[6]='C' = P[4]'C' → 匹配, j 到达 m=5
07
匹配成功! 返回起点 = i − m = 7 − 5 = 2
关键观察
第 03 步是 KMP 的灵魂——失配后 i 没有回退, 而朴素算法此时会把 i 退到 1 重新开始。
省下的就是这部分回退的代价, 才换来 O(n+m) 的总复杂度。
